1. Resumen Ejecutivo

Rosé Cafe Bar — Sistema de Gestión de Reservas es una aplicación web de página única (SPA) desarrollada para administrar las reservaciones del restaurante Rosé Cafe Bar. La aplicación resuelve dos necesidades fundamentales:

  • Para el cliente: Un formulario público, elegante y optimizado para móviles donde puede solicitar una reserva en menos de 30 segundos.
  • Para el administrador: Un panel de control protegido con dashboard de métricas, gestión completa de reservas (CRUD) y confirmación directa por WhatsApp.

Objetivos Clave

  • Reducir los No-Shows mediante confirmación vía WhatsApp
  • Eliminar la gestión manual telefónica de reservas
  • Proveer métricas operativas en tiempo real al administrador
  • Experiencia bilingüe (Español / Inglés) para clientes
  • Interfaz premium con modo oscuro, animaciones y diseño responsive

5. Frontend — Componentes y Vistas

La aplicación consta de 6 componentes React, cada uno con su propio archivo CSS modular.

5.1 LandingPage — Página Pública

Ruta: /

  • Hero visual con imagen de fondo, overlay degradado y blur
  • Logo + branding de Rosé Cafe Bar
  • Toggle de idioma ES/EN en el header
  • Formulario de reserva con campos: nombre, teléfono WhatsApp, fecha (DateSlider), hora, número de personas
  • Selector de personas visual con botones 1-6 y opción "7+" con prompt
  • Modal de confirmación con resumen de datos antes de enviar
  • Pantalla de éxito post-envío con indicador de estado "Pendiente"

5.2 DateSlider — Selector de Fecha Táctil

Componente de scroll horizontal que muestra los próximos 30 días como tarjetas seleccionables. Optimizado para dispositivos táctiles con scroll-snap.

5.3 Login — Acceso Administrativo

Ruta: /login

  • Formulario de credenciales con logo del restaurante
  • Manejo de errores con mensajes visuales
  • Botón de "Volver a la página principal"
  • Credenciales: admin / admin

5.4 Dashboard — Panel de Métricas

Ruta: /admin (pestaña Dashboard)

  • 3 tarjetas de KPIs: Reservas de hoy, del mes y del trimestre
  • Gráfico de área (Recharts) con actividad de los últimos 7 días
  • Panel "Próximas Hoy": Lista cronológica de reservas pendientes para el día actual

5.5 Reservations — Gestión CRUD

Ruta: /admin (pestaña Reservaciones)

  • Tabla completa con columnas: Cliente, Contacto, Fecha, Hora/Personas, Estado, Acciones
  • Barra de búsqueda en tiempo real por nombre o teléfono
  • Badges de estado: Confirmada Pendiente
  • Acciones por reserva:
    • ✅ Confirmar + enviar WhatsApp automático con mensaje personalizado
    • ✏️ Editar (modal con formulario pre-llenado)
    • 🗑️ Eliminar (con confirmación)
  • Modal de creación/edición con campos: nombre, teléfono, fecha, hora, personas

5.6 Sidebar — Navegación del Admin

  • Logo y branding
  • Navegación: Panel Interno / Reservaciones (con indicador de tab activo)
  • Footer: Configuración y Cerrar Sesión

7. Enrutamiento y Autenticación

7.1 Rutas

RutaComponenteAccesoDescripción
/LandingPagePúblicoFormulario de reservas para clientes
/loginLoginPúblicoFormulario de autenticación
/admin/*AdminLayoutProtegidoPanel administrativo (Dashboard + Reservas)

7.2 Componente ProtectedRoute

HOC que verifica isAuthenticated() antes de renderizar las rutas de admin. Redirige automáticamente a /login si no hay sesión activa.

7.3 Flujo de Autenticación

Cliente pulsa "Admin Panel" → /login → Ingresa credenciales
  → loginAdmin("admin","admin") → localStorage["restaurant_auth_token"] = "authenticated"
  → Navigate → /admin → ProtectedRoute verifica → AdminLayout renderiza

Cerrar Sesión → logoutAdmin() → Remove token → Navigate → /

8. Internacionalización (i18n)

La landing page soporta dos idiomas con un toggle visual ES | EN. El panel de admin permanece exclusivamente en español.

  • Implementación: Diccionario JSON en translations.js con 26 claves
  • Estado: useState('es') en LandingPage
  • Uso: Función t(key) como wrapper de getTranslation(lang, key)
  • Fallback: Si una clave no existe en el idioma seleccionado, retorna la versión en inglés
  • Localización de fechas: toLocaleDateString('es-ES') / toLocaleDateString('en-US')

12. Guía de Desarrollo y Despliegue

12.1 Comandos Disponibles

ComandoDescripción

12.2 Pruebas en Dispositivo Móvil

Se configuró localtunnel para exponer el servidor local a internet y probar en dispositivos físicos durante desarrollo. El host pretty-dodos-repair.loca.lt está permitido en vite.config.js.

12.3 Despliegue

El build de producción (npm run build) genera archivos estáticos en /dist que pueden desplegarse en cualquier hosting estático: Vercel, Netlify, GitHub Pages, Firebase Hosting, etc.
